Test cases can be of type:

  • True Positive (TP): Real vulnerabilities that should be reported by the tool. The SARD suite expects the corresponding CWE (Common Weakness Enumeration) to be reported for this vulnerability. We only measure on a per-sourcefile basis: one or more TP expected vs. one or more result from Kiuwan. We do not compare source code line numbers or number of TP's expected in a file.
  • True Negative (TN): Test cases where specifically no vulnerability should be reported. These cases cannot be detected, since they occur in the same source code file where a TP is reported. The SARD suite does refer to a source code line number where no vulnerability should be detected, but this is not sufficiently accurate to compare with line numbers reported by Kiuwan.

Definitions:

  • False Negative (FN): When a tool fails to detect a TP: The SARD suite expects a vulnerability of a certain category/CWE, but the tool does not report it.
  • Score = True Positive Rate (TPR) = TP / (TP + FN)
